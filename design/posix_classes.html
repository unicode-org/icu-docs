Recommendations for POSIX-Style Properties

Last updated: 2003-04-03, MED

The POSIX-style property names are are not well specified, and don't really map well to the broader types of characters available in Unicode/10646. For example, there is no provision for titlecase. However, many programs use them, so for compatibility it is best to come up with uniform set of recommendations for how they should be interpreted in a Unicode context. The following compares Perl, ICU, and Java, and tries to derive a recommendation. Feedback is welcome.

Notes:

  • The Java function names are methods on Character, unless otherwise noted. Note that the Java functions use out-of-date properties and/or definitions, especially in the cases marked *.
  • The normal property abbreviations are used, e.g. gc for General_Category.
  • For values of the General Category, see General_Categories.
  • HT is U+0009 <CHARACTER TABULATION (HT)>
  • Broken Links: some of the following links may not work, since they are to beta versions. If so, see the Notes.
