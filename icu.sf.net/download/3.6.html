Download ICU 3.6 d02 (beta) release

Description

This the d02 draft version of a major ("reference") release with new features and new and modified APIs.

Major changes in ICU 3.6 will include the following:

  • A new charset detection API has been added
  • Support for Unicode 5.0, UCA 5.0 and CLDR 1.4 have been added
  • Default timezone detection on Unix machines has been improved
  • Dictionary based detection of Thai word boundaries is now active for all locales
  • New extensible framework for dictionary based word boundaries
  • The BreakIterator API now uses an abstract text interface called UText
  • UText abstract text interface enhanced for 64 bit indexing, read-only locking, better performance
  • Tibetan font layout support as been added
  • The BiDi algorithm has been enhanced to be more flexible and efficient
  • The timezone data is now easy to update when ICU is downloaded from CVS
  • A new icupkg tool has been created for easier management of .dat data archives
  • Performance
    • The u_sprintf/u_sscanf performance from the icuio library has been improved for number formatting/parsing
    • Opening and closing a charset converter is significantly faster

Source Code Download

In addition to these .tar and .zip files, the code is also tagged in CVS with release-3-6-d02.

The ICU4C 3.6 d02 source downloads contains a pre-built .dat data archive with ICU's data rather than the data source files. This is to simplify the build process for the majority of users and to reduce platform porting issues. If you need the data source files for customization, then please download the ICU source code from CVS.

File Size Description
icu4c-3_6_d02-src.zip 10.6 MB ZIP file for Windows platforms
icu4c-3_6_d02-src.tgz 9.8 MB gzipped tar archive for Unix and other platforms

Platform Installation Notes

Please be sure to view the readme.html that is included in ICU. Most installation questions can be answered there. The z/OS (OS/390) unpax-icu.sh script is required to unpackage the compressed tar file on z/OS, and the i5/OS (OS/400) unpax-icu.sh script is required to unpackage the compressed file on i5/OS.
