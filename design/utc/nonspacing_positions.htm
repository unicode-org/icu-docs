From: Mark Davis
Date: 2005-05-09

While doing some work on confusables, I found it useful to categorize all the non-spacing marks by position. The combining character class can be used for this purpose, but needs to be extended to the characters with zero or fixed position classes.

1. I would like feedback on this categorization, to make sure I have things in the right places, since it is often difficult ( especially for Indic ) to figure out from the code chart; especially those characters that are attached, or that should be to the right or left ( eg Above_Right instead of Above ). The position can affect whether or not we call two characters confusable.
