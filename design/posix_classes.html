Recommendations for POSIX-Style Properties

Last updated: 2003-04-03, MED

The POSIX-style property names are are not well specified, and don't really map well to the broader types of characters available in Unicode/10646. For example, there is no provision for titlecase. However, many programs use them, so for compatibility it is best to come up with uniform set of recommendations for how they should be interpreted in a Unicode context.

Feedback is welcome.

Perl POSIX Recommendations with Unicode Comments
  alpha [\p{Alphabetic} \p{General_Category=Mark}] Alphabetic includes more than General_Category = Letter. Note that marks (Me, Mn, Mc) are required in many languages. It is a simplification to include them, since they could be applied to non-alphabetics, but excluding them would cause many words to fail the isAlpha test. See DerivedCoreProperties-4.0.0d20b.txt for Alphabetic, also DerivedGeneralCategory-4.0.0d19b.txt
  alnum [\p{alpha}\p{digit}] Simple combination of other properties
  blank [[\p{Whitespace} - {cntl}]
\N{CHARACTER TABULATION} ]		 "horizontal" whitespace. See PropList-4.0.0d20b.txt for WhiteSpace.
  cntrl [\p{General_Category=Control} \p{General_Category=Line_Separator} \p{General_Category=Paragraph_Separator}] Does not include Format, since some of those are needed in the representation of normal text. However, includes the Zl, Zp, since they are very control-like.
\d digit [\p{Number_Type = Decimal}] Non-decimal numbers (like Roman numerals) are normally excluded. In U4.0+, this is the same as General_Category = Decimal_Number (Nd). See DerivedNumericType-4.0.0d19b.txt
  graph [^\p{cntl}\p{space}] Simple combination of other properties
  lower [\p{Lowercase}] Lowercase includes more than General_Category = Lowercase_Letter (Ll). See DerivedCoreProperties-4.0.0d20b.txt.
  print [^\p{cntrl}] Simple combination of other properties
  punct [\p{General_Category=Punctuation}]  
\s space [\p{Whitespace}]  
  upper [\p{Uppercase} \p{General_Category=Titlecase_Letter}] Uppercase includes more than General_Category = Uppercase_Letter (Lu). The titlecase letters (Lt) are added because although neither fish nor fowl, adding them will probably work better for old software. See DerivedCoreProperties-4.0.0d20b.txt.
\w word [\p{alpha} \p{digit} \N{LOW LINE}] This is only an approximation to Word Boundaries (See UAX #29: Text Boundaries). The digits and "_" are lumped in for compatibility.
  xDigit [\p{Hex_Digit}] See PropList-4.0.0d20b.txt
\X   Default Grapheme Cluster See UAX #29: Text Boundaries, also GraphemeClusterBreakTest.html
\b   Default Word Boundaries If there is a requirement that /b align with \w, then it would use the approximation above instead. See UAX #29: Text Boundaries, also WordBreakTest.html.
