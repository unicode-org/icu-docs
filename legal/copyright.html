ICU Copyrights

The ICU project files contain code with several kinds of copyrights. They fall into several categories:

Copyright <date> IBM and others
This is the standard copyright notice that was specified by the IBM Public License. All IBM-developed code in ICU4C and ICU4J will contain this notice. In the vast majority of cases — even where they say "and others" — files do not actually have any other contributions other than from IBM. See Code Contributions.

Copyright <date> <contributor> and others
Outside developers or companies who are allowed to contribute code to ICU may have their own copyright statements. However, all material contributions are covered by a copyright agreement. See Code Contributions.

Copyright <date> Free Software Foundation
ICU4C (not ICU4J) contains three build scripts with FSF copyrights: configure, config.sub, and config.guess. This often causes confusion, because most FSF code is covered by the GNU license, which is viral. These three files, however, contain explicit statements that they are not covered by the GPL when distributed as part of other projects, and that they may be redistributed under other licenses.  For details, see FSF.

Copyright <date> Massachusetts Institute of Technology
ICU4C (not ICU4J) contains one file, install-sh, with an MIT copyright. However, the file includes an explicit header saying that it can be redistributed without fee. For details, see MIT.

Data Files

# Copyright (c) <date> Unicode, Inc.
As of Unicode 4.0.1, the license terms have been made more explicit on the files that we redistribute from the Unicode Character Database. This permitted is under the terms of the license. For details, see UCD.

A. Free Software Foundation

Applies to ICU4C. Does not apply to ICU4J.

The statements for two of the files under "Free Software Foundation" are the following. Note that we do "distribute this file as part of a program that contains a configuration script generated by Autoconf" and therefore we "include it under the same distribution terms that you use for the rest of that program" (see bold quoted text below).

Further details: Development of C/C++ code that is portable across a large number of platforms and compilers requires adaptation to the many differences between them. The Autoconf tool achieves this for ICU and for many other development projects. It needs these script files in order to function. The files themselves, as published by their authors and quoted below, contain an exception to the GPL that allows commercial (non-GPL) software development using these files. The exception requires the distribution of the files. These files are part of the build setup, not part of the library source code. Shipping build files is only relevant for source downloads. ICU4C fulfills the condition by including the required files in its source download.

These files are not used for Java development and therefore do not apply to ICU4J.
